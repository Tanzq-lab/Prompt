# Figma 初始化提示词｜分阶段接力使用说明

本拆分包基于《Figma初始化.md》整理，目标是避免一次性输出 Page00 / Page01 / Page02 / Page03 导致质量下降。

## 使用顺序

1. 先发送 `01_第0阶段_总控Agent_需求追问.md`，并附上你的具体游戏需求。
2. 当 GPT 完成追问并达到 95% 理解信心后，它只会输出项目理解摘要、项目边界和 Page00 执行前提。
3. 然后发送 `02_第1阶段_Page00运行框架设计师.md`，让 GPT 只输出 Page00 运行框架。
4. 你确认 Page00 后，再发送 `03_第2阶段_Page01视觉规范设计师.md`。
5. 你确认 Page01 后，再发送 `04_第3阶段_Page02页面UI设计师.md`。
6. 你确认 Page02 后，再发送 `05_第4阶段_Page03原子资源设计师.md`。
7. 你确认 Page03 后，再发送 `06_第5阶段_最终质量检查与交付说明.md`。

## 核心规则

- 不得一次性生成所有 Page。
- 每个阶段只解决当前阶段的问题。
- 后续阶段可以标注为“待处理”，但不能提前完整生成。
- 每个阶段结束后必须等待用户确认。
- Page01 / Page02 / Page03 必须由用户主动发送对应阶段提示词后才能继续。
- 后一阶段必须继承前一阶段已确认结果，不得重新脑补或推翻。
- 如果用户修改前一阶段内容，后续阶段必须同步更新依据。

## 文件说明

| 文件 | 用途 |
|---|---|
| 01_第0阶段_总控Agent_需求追问.md | 需求追问、边界确认、达到 95% 理解信心 |
| 02_第1阶段_Page00运行框架设计师.md | 输出 Page00：运行框架、页面范围、用户路径、Page03 资源预拆分草稿 |
| 03_第2阶段_Page01视觉规范设计师.md | 输出 Page01：视觉规范、色彩、字体、图形语言、视觉规则 |
| 04_第3阶段_Page02页面UI设计师.md | 输出 Page02：页面 UI、页面存在理由、交互状态、Page03 拆分依据 |
| 05_第4阶段_Page03原子资源设计师.md | 输出 Page03：组件、导出资源、Holopix 图片优化标注 |
| 06_第5阶段_最终质量检查与交付说明.md | 输出最终检查清单和交付说明 |
| 99_原始完整版_备份.md | 原始完整文档备份，方便对照 |

## Holopix 边界

- Holopix 只作为图片素材生成、替换、优化来源。
- Figma 负责页面结构、文字排版、按钮状态、页面适配、命名、切图和可交付系统化整理。
- Page03 只标注哪些图片类资源适合交给 Holopix 优化，不要求输出 Holopix 具体 Prompt。
