# Workflow

请严格按照以下流程执行：

## 1. 读取文件

如果我上传的是单张图片：

1. 读取图片
2. 判断是否有透明通道
3. 检查透明边缘是否存在白边、灰边或浅色污染

如果我上传的是压缩包：

1. 解压项目或素材包
2. 遍历图片资源
3. 只筛选带透明通道的 PNG / WebP
4. 记录原始路径、分辨率、文件大小、是否存在疑似白边

---

## 2. 判断白边类型

请将问题分成以下几类：

1. 明显白边：边缘存在一圈白色或浅色描边
2. 半透明白污染：Alpha 边缘像素混入白色，换深色背景时明显发白
3. 灰边 / 脏边：边缘有灰色、浅色残留
4. 抠图毛刺：边缘锯齿、碎点、半透明杂点
5. 不建议处理：边缘本身是角色高光、白色衣服、白色头发、白色道具，不能误删

---

## 3. 执行处理

优先使用以下非重绘方式：

### A. 去白色杂边

适合原图来自白底抠图的素材。

处理逻辑：

1. 找到 Alpha 边缘区域
2. 检测半透明白色污染像素
3. 将边缘 RGB 从白色污染中拉回到邻近不透明像素颜色
4. 保留原 Alpha 层级

---

### B. 轻微 Defringe

适合边缘有 1px 到 2px 明显白边的素材。

处理逻辑：

1. 只处理透明边缘附近的 1px 到 2px
2. 用邻近主体颜色替换白边颜色
3. 不扩大、不缩小主体轮廓

---

### C. Alpha 轻微收缩

适合边缘白边均匀但不涉及头发丝的素材。

处理逻辑：

1. Alpha 边缘轻微向内收缩
2. 默认不超过 1px
3. 必要时加 0.2px 到 0.5px 的边缘过渡
4. 避免角色轮廓明显变瘦

---

### D. 细节保护处理

适合头发丝、毛发、手指、衣服细边、半透明阴影等复杂边缘。

处理逻辑：

1. 不粗暴收缩
2. 优先做颜色去污染
3. 保留 Alpha 层级
4. 如无法完全清理，宁可保留少量边缘，也不要破坏细节

---

## 4. 质量检查

处理后请检查：

1. 原图尺寸是否完全一致
2. 透明背景是否仍然透明
3. 主体是否没有明显变形
4. 边缘白边是否减少
5. 是否出现黑边、灰边、新毛刺
6. 是否误伤白色角色、白衣服、白色道具、高光区域
7. 在深色背景下预览是否明显改善
8. 在浅色背景下预览是否没有产生新问题

---

## 5. 输出文件

如果是单张图：

1. 输出处理后的透明 PNG
2. 文件名建议添加 `_clean` 后缀
3. 保持原始分辨率

如果是批量压缩包：

1. 输出一个处理后的补丁包
2. 保持原始目录结构
3. 只包含被处理或需要覆盖的图片
4. 不要在压缩包里额外包一层无意义的顶级目录
5. 同时输出处理报告
