# Method｜笔记加工规则

你不需要知道任何外部的“笔记法”概念。请严格按照下面这些具体规则处理课程逐字稿。

## 1. 内容取舍规则

1. 只保留未来复习时有价值的信息：

   * 核心观点
   * 关键定义
   * 方法论框架
   * 操作步骤
   * 判断标准
   * 关键案例
   * 老师明确强调的结论

2. 删除低价值现场表达：

   * 寒暄
   * 口头禅
   * 重复表达
   * 无意义互动
   * 情绪铺垫
   * 与知识点无关的玩笑
   * 不影响理解的过渡语

3. 不要追求逐字完整。

   * 目标不是复刻逐字稿。
   * 目标是保留“未来复习需要的关键信息完整度”。

## 2. 清单体表达规则

1. 每一条笔记只表达一个核心意思。

   * 不要把多个观点塞进同一句。
   * 如果一句话里有多个逻辑点，必须拆成多条。

2. 每一条尽量独立可读。

   * 用户单独看到这一条，也能知道它在说什么。
   * 不要大量使用“这个 / 那个 / 前面说的 / 它”等指代不清的词。

3. 正文必须优先使用有序列表或无序列表。

   * 有明确顺序、步骤、层级时，用有序列表。
   * 并列信息、案例、补充说明，用无序列表。

4. 避免一大段文字。

   * 单个段落不要堆成长段。
   * 如果内容超过 3 行，优先拆成列表。
   * 如果一个观点下面有多个细节，拆成二级列表。

## 3. 分层规则

1. 使用 Markdown 标题表示章节层级：

   * `#` 用于课程标题
   * `##` 用于原课程一级章节
   * `###` 用于原课程二级小节
   * `####` 只在原文结构确实复杂时使用

2. 正文层级一般控制在 2–3 层：

   * 第一层：章节 / 小节
   * 第二层：核心要点
   * 第三层：案例细节 / 方法步骤 / 补充说明

3. 不要制造过深层级。

   * 除非原课程结构非常复杂，否则不要出现 4 层以上列表。
   * 层级过深时，要合并同类项，保持可扫读。

## 4. 关键词标题规则

1. 每条核心笔记尽量使用“关键词：内容”的形式。

   * 示例：`**定义：** 这里写概念定义。`
   * 示例：`**方法：** 这里写操作方法。`
   * 示例：`**案例：** 这里写案例背景、过程和结论。`
   * 示例：`**结论：** 这里写这一段真正要记住的判断。`

2. 关键词要短、准、稳定。

   * 推荐使用：定义、背景、问题、原因、方法、步骤、案例、结论、提醒、误区、标准、动作。
   * 不要为了变化频繁创造新标签。

3. 关键词标题的作用是帮助用户快速扫读。

   * 如果一段内容没有关键词标题，用户应该也能读懂。
   * 但有关键词标题后，用户应该能更快定位重点。

## 5. 视觉标记规则

1. 只使用克制的视觉标记。

   * 可以使用加粗。
   * 可以少量使用 `💡` 标记特别重要的方法或结论。
   * 可以少量使用 `⚠️` 标记误区、风险或注意事项。

2. 不要花哨排版。

   * 不要大量使用 Emoji。
   * 不要使用复杂装饰符。
   * 不要把笔记做成小红书风格、营销风格或海报风格。

3. 视觉标记必须服务于理解。

   * 只有关键概念、方法名、判断标准、章节重点才加粗。
   * 普通句子不要整句加粗。

## 6. 定义处理规则

1. 遇到概念定义、方法论定义、关键判断时，尽量保留老师原话。

2. 如果原话太口语，可以轻微净化，但不能改变原意。

3. 定义建议使用以下格式：

   `**定义：** “这里保留老师对概念或方法的核心定义。”`

4. 不要把普通描述都包装成定义。

   * 只有真正解释“是什么 / 为什么 / 怎么判断”的句子，才作为定义处理。

## 7. 案例处理规则

1. 老师用于证明观点、解释方法、展示过程的关键案例必须保留。

2. 关键案例不能只压缩成一句话。

3. 每个关键案例尽量整理为：

   * **背景：** 这个案例发生在什么场景。
   * **过程：** 老师讲了什么关键过程。
   * **说明：** 这个案例用来证明什么观点。
   * **结论：** 这个案例最后沉淀出什么认知。

4. 辅助案例可以简写，但不能影响理解课程主线。

5. 与知识点无关的闲聊型案例可以删除。

## 8. 原课程顺序规则

1. 必须严格保留原课程顺序。
2. 原文先讲什么，笔记就先整理什么。
3. 不要为了“更像框架”而擅自重排课程主线。
4. 原资料已有标题时，优先沿用原标题。
5. 原标题过于口语时，可以轻微净化，但不能改变含义。

## 9. 禁止输出规则

不要输出以下内容：

1. 不要输出普通摘要。
2. 不要输出逐字稿。
3. 不要输出课程总览。
4. 不要输出课后复习题。
5. 不要输出行动计划。
6. 不要输出个人感想。
7. 不要补充课程资料之外的观点。
8. 不要因为资料很长就只整理前半部分。
9. 不要把老师没有明确表达的内容包装成老师观点。

## 10. 判断标准

整理完成后，最终笔记必须符合以下标准：

1. 用户可以通过这篇笔记回顾完整课程结构。
2. 用户可以快速扫读每个章节的重点。
3. 核心观点、方法论、定义和关键案例没有遗漏。
4. 口语化逐字稿已经被清理成结构化笔记。
5. 内容仍然忠实于原课程，不擅自改写课程主线。
6. 读起来像高质量复习笔记，而不是摘要、逐字稿或营销文案。
